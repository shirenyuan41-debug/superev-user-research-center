# 变更记录

## 2026-04-18 至 2026-04-19

### 架构与运行方式

- 项目从“前端原型”演进为“前端 + Express 后端 + MySQL + 本地文件存储”的联调架构
- 根命令 `npm run dev` 改为同时启动前端和后端
- 新增 `backend/`、`shared/`、`scripts/` 目录作为后端、共享逻辑和启动辅助层
- 新增 `启动本地部署.command`，支持在 macOS 桌面环境双击启动本地联调

### 登录与会话

- 登录改为走真实后端接口，不再只是前端伪登录
- 登录态恢复改为依赖 `GET /api/auth/me`
- 会话改为 `HttpOnly Cookie + express-session + MySQL`
- 连续 12 小时无交互自动退登

### 用户研究 > 访谈分析

- 文档上传、文档列表、文档编辑、文档删除改为走真实后端
- 原始文件会落盘到 `backend/storage/source-documents/YYYY/MM/`
- 当前真实分析格式收敛为 `docx/txt`
- 同一业务线下，同名文件会自动去重，只保留最新一份
- 分析服务收敛为“两段式”：
  - 第一步生成 `summary / insights / journey / persona`
  - 第二步基于洞察生成 `actions`
- 分析结果会记录实际使用的 Prompt 版本、模型提供方和模型名

### 用户研究 > 用户画像

- 页面改为直接消费“用户研究”模块的真实分析结果
- 不再依赖静态样例和随机画像信息
- 单体 / 聚合画像会根据当前受访对象选择数量自动切换
- 左侧画像卡改为事实驱动展示
- 旅程图改为结构化双排展示

### 用户研究 > 使用反馈

- 访谈分析页里的赞/踩和文字反馈改为先在当前页暂存
- 只有点击“确认并同步”后才会写入反馈中心
- 反馈结构扩展为：
  - 反馈原声
  - AI 评价总结
  - AI 优化建议
  - 来源文档

### Todo 管理

- Todo 改为走真实后端持久化
- Todo 可记录来源模块、来源文档和来源业务线快照

### 系统管理 > 提示词管理

- Prompt 从单一全局内容演进为多模块版本治理
- 当前支持模块：
  - 用户研究
  - 销转研究
  - 行业研究
  - 舆情研究
  - 员工研究
- 当前支持状态：
  - 草稿
  - 已发布
  - 已归档

### 系统管理 > 数据入口

- 用户研究源文档台账改为真实后端数据源
- 列表支持预览、编辑、删除
- 同名源文档自动去重

### 数据迁移

- 新增旧版 localStorage 导出页 `?page=legacy-export`
- 新增后端导入脚本 `import-legacy-json.ts`
- 旧版文档、反馈、Todo 和 Prompt 版本可以导入 MySQL

### 文档同步

- README、PRD、设计文档、提示词文档、背景知识文档已整体同步到当前前后端联调版状态

## 2026-04-17 至 2026-04-18

### 用户研究 > 用户画像

- 业务线筛选器改为当前页固定选项：`全部 / 超级订阅 / 灵活订阅 / 其他`
- 受访对象筛选器改为读取“用户研究 > 访谈分析 > 业务线历史”中已解析文档，并从文件名 `-` 后提取客户名
- 受访对象选中数量与画像模式联动：
  - 选中 `0` 个或 `1` 个时默认展示“单体画像”
  - 选中 `2` 个及以上时默认展示“聚合画像”
- 左侧画像信息卡不再展示随机内置头像，改为展示与当前受访对象逐字稿相对应的事实摘要、事实标签和事实列表
- “用户决策旅程图”改为结构化两排展示：
  - 左侧固定行标题为“用户心路”和“用户VOC”
  - 右侧按阶段横向对齐两排浅灰色信息框
  - 整体宽度控制在当前卡片一屏内展示
