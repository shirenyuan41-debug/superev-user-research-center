# 产品需求文档（PRD）

## 1. 文档信息

- 产品名称：SUPEREV 用研中心
- 当前阶段：MVP 前后端联调版
- 当前重点：把“用户研究”主闭环从原型推进到真实后端承载
- 对应代码状态：前端 + Express 后端 + MySQL + 本地文件存储

## 2. 产品背景

“超级电动”是一家新能源电车订阅平台，核心业务包括“超级订阅”和“灵活订阅”。用户研究、销转研究、反馈归因和跨部门跟进，本质上都依赖对非结构化材料的高质量处理，但传统工作方式存在几个明显问题：

- 研究资料分散在文档、聊天记录和个人经验里
- 洞察提炼严重依赖人工整理，效率低
- AI 结果质量好坏难追踪，Prompt 不好持续优化
- 研究结论经常停留在报告里，不能顺滑转成行动项
- 历史资料难沉淀、难复用、难复盘

因此需要一个内部研究工作台，把“上传资料、调用 AI、展示结构化结果、收集反馈、沉淀 Todo、维护 Prompt 版本”串成闭环。

## 3. 产品定位

SUPEREV 用研中心是一个服务内部团队的 AI 研究运营台。

它不是面向外部客户的产品，也不是只做报告展示的页面集合，而是一个帮助研究团队持续生产、持续验证、持续沉淀的内部工作系统。

## 4. 目标用户

### 4.1 核心用户

- 用户研究负责人 / 研究员
- 销售研究与转化分析人员
- 产品经理、设计师、运营、客服等研究结果消费方

### 4.2 次级用户

- 管理员：管理登录账号、Prompt 模块、数据入口
- HR / 组织发展团队：未来使用员工研究能力

## 5. 当前版本目标

### 5.1 业务目标

- 把“用户研究”主闭环落到真实后端
- 让文档、分析结果、反馈、Todo 和 Prompt 版本脱离浏览器本地存储
- 让研究结果可以被持续追踪和复用
- 为后续销转研究、舆情研究、行业研究和员工研究铺好基础设施

### 5.2 产品目标

- 提供稳定的登录态和后端数据持久化
- 让文档上传、分析、反馈、Todo、Prompt 管理可在一套系统内流转
- 保持页面结构清晰，继续支撑演示、评审和后续开发

### 5.3 当前非目标

- 当前不做细粒度 RBAC 权限控制
- 当前不做多租户隔离
- 当前不做完整的 BI 分析平台
- 当前不让所有研究模块都接入真实模型

## 7. 核心业务闭环

## 7.1 用户研究主闭环

1. 用户登录系统
2. 上传研究文档
3. 后端解析文本并保存源资料
4. 调用当前模块已发布 Prompt 触发分析
5. 前端展示速览、洞察、旅程图、画像、行动建议、完整报告
6. 用户提交赞/踩反馈并同步到反馈中心
7. 用户把行动建议转成 Todo
8. Prompt 管理页根据反馈继续迭代版本

## 7.2 旧版数据迁移闭环

1. 开发环境打开 `legacy-export` 页面导出旧版本地数据
2. 生成 JSON 包
3. 使用后端导入脚本写入 MySQL
4. 后续统一从新后端继续使用

## 8. 功能需求

## 8.1 登录与会话

### 目标

- 让系统具备真实会话能力，不再依赖前端伪登录

### 需求

- 支持账号和密码登录
- 支持刷新页面后恢复登录状态
- 用户连续 12 小时无交互自动退登
- 支持主动退出登录
- 当后端不可用时，登录页明确提示问题来源

### 当前状态

- 已接真实后端登录
- 已接 `express-session`
- 当前仍只有基础登录，不包含复杂权限控制

## 8.2 访谈分析

### 目标

- 将研究文档快速转成结构化结果，并把结果沉淀到系统里

### 输入

- `docx`
- `txt`

### 当前真实限制

- `pdf/png/jpg/jpeg` 暂不走真实分析
- 前端会明确提示当前真实分析暂只支持 `docx/txt`

### 核心功能

- 选择业务线
- 上传单个文件
- 保存原始文档
- 调用后端分析
- 展示 6 个结果页签：
  - 速览
  - 洞察
  - 旅程图
  - 画像
  - 行动建议
  - 完整报告
- 导出报告
- 提交反馈
- 加入 Todo
- 浏览历史文档

### 当前状态

- 已接真实后端
- 已接真实 Prompt 版本读取
- 当前是系统里最完整的可用模块

## 8.3 用户画像

### 目标

- 直接复用访谈分析结果，减少二次整理

### 需求

- 业务线筛选
- 时间范围筛选
- 受访对象筛选
- 单体画像 / 聚合画像切换
- 决策旅程展示
- 事实卡、标签、JTBD、关键需求、主要阻碍、跟进建议展示

### 当前状态

- 已与“用户研究”分析结果联动
- 已不再依赖静态样例
- 仍属于前端结果消费页，没有独立后端画像生成服务

## 8.4 Todo 管理

### 目标

- 把研究结论转成可跟进的行动项

### 需求

- 新增 Todo
- 编辑 Todo
- 删除 Todo
- 状态更新
- 搜索与筛选
- 记录来源模块和来源文档快照

### 当前状态

- 已接真实后端
- 支持基础 CRUD

## 8.5 使用反馈

### 目标

- 把 AI 结果质量变得可观测、可沉淀、可反哺 Prompt

### 需求

- 记录赞/踩
- 记录反馈原声
- 生成 AI 评价总结
- 生成 AI 优化建议
- 支持按模块和业务线查看反馈

### 当前状态

- 已接真实后端
- 反馈必须通过“确认并同步”才会写入

## 8.6 Prompt 管理

### 目标

- 让不同研究模块的 Prompt 版本可管理、可发布、可回溯

### 需求

- 支持多模块
- 支持草稿 / 已发布 / 已归档
- 支持保存草稿
- 支持发布新版本
- 支持查看版本历史

### 当前状态

- 已接真实后端
- 已支持 5 个模块的版本管理
- 当前真正接入模型调用的仍是“用户研究”

## 8.7 数据入口

### 目标

- 管理外部连接器状态和用户研究源文档台账

### 需求

- 查看连接器状态
- 查看源文档列表
- 文档预览
- 文档删除
- 文档编辑
- 搜索和筛选

### 当前状态

- 用户研究源文档台账已接真实后端
- 外部连接器仍是原型展示

## 8.8 数据中心

### 目标

- 统一承接结构化结果、原始资料和搜索入口

### 当前状态

- 仍以静态原型为主
- 尚未接真实数据查询

## 8.9 销转研究 / 舆情 / 行业 / 员工研究

### 目标

- 逐步形成多模块研究平台

### 当前状态

- 页面结构已占位
- Prompt 模块已存在
- 尚未接真实分析链路

## 9. 核心数据对象

当前系统围绕以下对象运行：

- 用户 `users`
- 会话 `auth_sessions`
- Prompt 模块 `prompt_modules`
- Prompt 版本 `prompt_versions`
- 研究文档 `research_documents`
- 反馈 `feedbacks`
- Todo `todos`
- 历史导入批次 `legacy_import_batches`

## 10. 非功能需求

- 默认中文界面
- 本地开发可一键同时启动前后端
- 后端对核心对象做持久化
- 分析结果必须可回看、可复用、可追溯
- 会话稳定，不因短时刷新丢失登录态
- 文档原件可落盘保存

## 11. 当前版本成功标准

- 用户可以完成“登录 -> 上传 -> 分析 -> 反馈 -> 加 Todo”的完整链路
- Prompt 发布后，后续分析可以自动使用最新已发布版本
- 数据不再只存在浏览器本地
- 旧版 localStorage 数据可以导入新后端

## 12. 当前已知限制

- 当前真实分析只支持 `docx/txt`
- 当前只有“用户研究”接入真实分析
- 分析结果没有缓存机制
- 多数管理页和数据中心仍是展示原型
- 权限控制尚未真正落地

## 13. 后续迭代建议

### P1

- 为销转研究接真实分析
- 为数据中心接真实查询
- 为分析结果增加缓存

### P2

- 支持更多输入文件类型
- 引入异步任务和进度反馈
- 补齐权限和审计

### P3

- 打通外部数据连接器
- 建立统一资产关系图谱
- 支持跨模块洞察联动
